# Классификация изображений спортивной тематики с помощью сверточной нейронной сети

Курсовая работа по дисциплине «Нейронные сети и машинное обучение». Цель работы — построить и обучить сверточную нейронную сеть, способную классифицировать изображения по 100 различным видам спорта.

## Постановка задачи

Дано: набор фотографий, размеченных по 100 категориям спорта (от боулинга до бобслея). Требуется построить модель, которая по входному изображению предсказывает вид спорта, изображённый на нём.

Это задача **многоклассовой классификации изображений** — одно из классических применений сверточных нейронных сетей в компьютерном зрении.

## Датасет

Использован публичный датасет **Sports Classification** (автор gpiosenka, размещён на Kaggle):

| Подвыборка | Количество изображений |
|-----|-----|
| Обучающая (train) | 13 492 |
| Валидационная (valid) | 500 |
| Тестовая (test) | 500 |
| Всего классов | 100 |

Датасет имеет стандартную структуру `ImageFolder`: каждый класс лежит в отдельной подпапке, имя папки соответствует названию вида спорта.

```
sports-classification/
├── train/
│   ├── air hockey/
│   ├── archery/
│   ├── ...
│   └── wrestling/
├── valid/
└── test/
```

## Выбор архитектуры

### Почему ResNet50

В качестве базовой архитектуры выбрана **ResNet50** — глубокая сверточная сеть с остаточными связями (residual connections), предложенная He et al. в 2015 году. Обоснование выбора:

1. **Глубина и выразительность.** 50 слоёв позволяют выучить иерархию признаков от простых (рёбра, текстуры) до сложных (части объектов, целые объекты).
2. **Доступность предобученных весов.** В составе `torchvision.models` поставляются веса, обученные на ImageNet (1.2 млн изображений, 1000 классов), что критично для transfer learning при ограниченном объёме обучающих данных.
4. **Баланс точности и вычислительной стоимости.** ResNet50 показывает высокое качество при разумных требованиях к памяти и времени обучения. Более тяжёлые модели (ResNet152, EfficientNet-B7) при нашем размере датасета не дают существенного прироста.

### Transfer learning

Обучение модели с нуля на 13 тысячах изображений нецелесообразно: для такой задачи необходимы сотни тысяч примеров. Поэтому применён **transfer learning** — перенос знаний из модели, предобученной на ImageNet.

Идея: низкоуровневые признаки (края, цветовые градиенты, текстуры) универсальны и применимы к любым изображениям, в том числе спортивным. Их не нужно учить заново — достаточно адаптировать модель под новые классы.

## Архитектура решения

Модель состоит из двух частей:

```
Входное изображение (3, 224, 224)
        │
        ▼
┌─────────────────────────┐
│      Backbone           │
│   ResNet50 (frozen,     │
│   потом fine-tuned)     │
│                         │
│   23.5M параметров      │
└───────────┬─────────────┘
            │
            ▼
     Вектор признаков (2048)
            │
            ▼
┌─────────────────────────┐
│   Голова классификатора │
│                         │
│   Dropout(p=0.3)        │
│   Linear(2048 → 100)    │
│                         │
│   ~205K параметров      │
└───────────┬─────────────┘
            │
            ▼
   Логиты по 100 классам
```

### Замена финального слоя

Оригинальный ResNet50 заканчивается слоем `Linear(2048 → 1000)` под 1000 классов ImageNet. Этот слой заменён на собственную «голову»:

- **Dropout с вероятностью 0.3** — регуляризация, предотвращающая переобучение за счёт случайного зануления части активаций при обучении.
- **Linear(2048 → 100)** — линейный классификатор под 100 целевых классов.

## Стратегия обучения

Применена **двухстадийная стратегия** обучения, стандартная для transfer learning:

### Стадия 1. Обучение головы (эпохи 1-5)

- Backbone заморожен (`requires_grad = False` для всех слоёв, кроме `fc.`).
- Обучается только новая голова классификатора (~205 тыс. параметров).
- Learning rate: `1e-3` — большой, так как голова инициализирована случайно и должна быстро адаптироваться.
- Оптимизатор: AdamW.

Назначение этапа: голова должна научиться правильно интерпретировать предобученные признаки backbone и спроецировать их в пространство 100 классов спорта. Без этого этапа размораживание сразу всей сети приведёт к разрушению полезных предобученных весов из-за больших начальных градиентов.

### Стадия 2. Тонкая настройка (эпохи 6-15)

- Размораживаются все параметры модели (23.7 млн).
- Learning rate уменьшается на порядок: `1e-4` — мало, чтобы аккуратно подстроить ImageNet-фичи под спортивную доменную область, не разрушая их.
- Оптимизатор пересоздаётся с новыми параметрами.

### Планировщик learning rate

Используется **косинусный отжиг (Cosine Annealing)** — learning rate плавно убывает по косинусу от стартового значения до близкого к нулю за `T_max` эпох. Это даёт более стабильную сходимость в конце обучения, чем фиксированный или ступенчатый LR.

### Функция потерь и оптимизатор

- **CrossEntropyLoss** — стандартный выбор для многоклассовой классификации. Внутри объединяет log-softmax и negative log-likelihood.
- **AdamW** — модификация Adam с корректно реализованным weight decay. Сходится быстрее SGD без необходимости тщательного подбора momentum.

## Предобработка и аугментация данных

### Нормализация

Все изображения нормализуются по статистике ImageNet:

```
mean = [0.485, 0.456, 0.406]   # средние по каналам R, G, B
std  = [0.229, 0.224, 0.225]   # стандартные отклонения
```

Эти значения — статистика, посчитанная по 1.2 млн изображений ImageNet. Используются именно они (а не статистика спортивного датасета), потому что предобученные веса ResNet50 «настроены» именно на такой входной диапазон.

### Аугментация (только для train)

| Преобразование | Параметры | Назначение |
|-----|-----|-----|
| Resize | 224 × 224 | Приведение к входному размеру ResNet50 |
| RandomHorizontalFlip | p = 0.5 | Устойчивость к зеркальному отражению |
| RandomRotation | ±15° | Устойчивость к небольшим поворотам |
| ColorJitter | brightness/contrast/saturation = 0.2 | Устойчивость к условиям освещения |
| ToTensor | — | PIL → tensor, нормирование в [0, 1] |
| Normalize | ImageNet mean/std | Центрирование вокруг нуля |

Для валидации и теста применяется только `Resize → ToTensor → Normalize` без случайных преобразований — метрики должны быть детерминированы.

## Гиперпараметры

| Параметр | Значение | Обоснование |
|-----|-----|-----|
| Размер батча | 64 | Компромисс между скоростью и стабильностью градиентов |
| Эпох всего | 15 | По логам обучения уже к 10-й эпохе модель сходится |
| Эпох прогрева (только голова) | 5 | Достаточно для адаптации линейного классификатора |
| LR головы | 1e-3 | Стандартный стартовый LR для AdamW |
| LR backbone | 1e-4 | На порядок меньше, чтобы не разрушить ImageNet-фичи |
| Размер изображения | 224 × 224 | Исходное разрешение, под которое обучен ResNet50 |
| Dropout | 0.3 | Умеренная регуляризация в голове |
| Seed | 42 | Воспроизводимость результатов |

## Результаты

### Метрики качества

| Метрика | Значение |
|-----|-----|
| Лучшая validation accuracy | **0.986** |
| Test accuracy | ~0.98 |

### Динамика обучения

```
Эпоха  | Train acc | Val acc | Время
─────────────────────────────────────
  1    |   0.592   |  0.878  |  59 с    стадия 1: голова
  2    |   0.840   |  0.912  |  44 с
  3    |   0.872   |  0.934  |  47 с
  4    |   0.898   |  0.924  |  46 с
  5    |   0.914   |  0.934  |  46 с
─────────────────────────────────────  размораживание backbone
  6    |   0.945   |  0.962  | 138 с    стадия 2: fine-tuning
  7    |   0.979   |  0.972  | 139 с
  8    |   0.990   |  0.980  | 138 с
  ...  |   ...     |  ...    |  ...
 14    |   0.999   |  0.986  | 138 с    лучшая модель
 15    |   0.999   |  0.984  | 138 с
```

Чётко видно качественный скачок после разморозки backbone (эпоха 6): val accuracy выросла с 0.934 до 0.962 за одну эпоху. Это подтверждает корректность двухстадийной стратегии.

Время на эпоху увеличилось с ~46 с до ~138 с, потому что после разморозки backbone градиенты считаются для всех 23.7 млн параметров.

### Сохранение лучшей модели

Каждую эпоху измеряется validation accuracy. Если она превышает текущий максимум — состояние модели (`state_dict`) сохраняется на диск как `best_model.pt`. На тесте загружается именно эта лучшая версия, а не последняя — это защищает от переобучения в финальных эпохах.

## Оценка модели

После обучения модель оценивается на отдельной тестовой выборке (500 изображений, не участвовавших ни в обучении, ни в валидации):

- **Test accuracy** — общая доля правильных предсказаний.
- **Classification report** (precision, recall, f1-score по каждому классу) — детальный разбор качества.
- **Confusion matrix** — матрица ошибок для визуализации того, между какими классами модель путается.

Пример инференса на одной картинке:

```
Файл: test/bobsled/5.jpg
Top-3 предсказания:
  bobsled            0.9999
  luge               0.0001
  formula 1 racing   0.0000
```

Модель не только правильно предсказывает класс, но и отдаёт высоко-вероятностный score, а на второе место помещает семантически близкий класс (luge — тоже сани).

## Выводы

Применение transfer learning на основе ResNet50 с двухстадийной стратегией обучения позволило достичь **98.6% accuracy** на валидации и сопоставимого качества на тесте при умеренных вычислительных затратах (около 30 минут на GPU). Ключевыми факторами успеха являются:

1. Использование предобученных на ImageNet весов вместо обучения с нуля.
2. Двухстадийная стратегия: сначала обучение головы при замороженном backbone, затем тонкая настройка всей сети с пониженным learning rate.
3. Аугментация данных, повышающая обобщающую способность модели.
4. Косинусный планировщик learning rate для стабильной сходимости.
5. Сохранение лучшей модели по метрике валидации, а не использование последней эпохи.
